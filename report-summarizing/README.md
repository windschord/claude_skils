# Report Summarizing スキル

調査結果や分析レポートを、4層の情報階層（エグゼクティブサマリー→概要→詳細分析→Appendix）で整理・再構成するスキルです。

## 概要

長大なドキュメントを意思決定者向けに読みやすく構造化します。段階的な情報開示（Progressive Disclosure）により、読者が必要な深さまで情報を取得できます。

## 主な機能

- **4層の情報階層**: サマリー→概要→詳細→Appendixで段階的に情報を開示
- **充実したエグゼクティブサマリー**: 結論（3-5行）、推奨事項、重要ポイント（各2-3行）
- **概要セクション**: PROS/CONS表、課題/対策表で全体像を把握
- **技術詳細の分離**: ソースコード修正案等はAppendixに配置
- **Mermaidダイアグラム**: マインドマップ、フローチャート等で全体像を可視化
- **内部リンク**: 各層間のナビゲーション
- **GitHub/VSCode対応**: 直接レンダリング可能なMarkdown形式

## 使用方法

### 既存ドキュメントの変換

```
このドキュメントを4層構造のレポート形式に変換してください
```

### 新規レポートの作成

```
調査結果をエグゼクティブサマリー形式でまとめてください
```

## ドキュメント構造（4層）

```
# レポートタイトル

## エグゼクティブサマリー（レベル1: 意思決定者向け）
├── 結論（3〜5行: 推奨理由、効果、トレードオフを含む）
├── 推奨事項（優先順位 + 理由）
├── 重要ポイント（各2-3行 + 概要/詳細へのリンク）
├── 全体像（Mermaidダイアグラム）
└── ナビゲーション（レベル別リンク）

## 概要（レベル2: 中間層）
├── [トピック1]の概要
│   ├── 要約（3-5行）
│   ├── PROS/CONS表（各1行）
│   ├── 課題/対策表（影響度付き）
│   └── 詳細へのリンク
├── [トピック2]の概要
└── 比較サマリー

## 詳細分析（レベル3: 本編）
├── 背景と目的
├── 調査範囲と方法
├── [トピック]の詳細分析
│   ├── PROS詳細
│   ├── CONS詳細
│   └── 課題と対策（実装方針、Appendix参照）
├── リスク分析
└── 次のステップ

## Appendix（レベル4: 技術詳細）
├── A. ソースコード修正案
├── B. 技術仕様
├── C. 用語集
└── D. 参考文献
```

## 各層の役割

| 層 | 対象読者 | 情報量 | 含める内容 |
|---|---------|--------|----------|
| サマリー | 意思決定者 | 1-2ページ | 結論、推奨理由、効果、トレードオフ |
| 概要 | 関係者 | 各0.5-1ページ | PROS/CONS一覧、課題/対策要約 |
| 詳細分析 | 実装担当者 | 必要に応じて | 詳細な分析、根拠、実装方針 |
| Appendix | 開発者 | 必要に応じて | ソースコード、技術仕様 |

## ベストプラクティス

1. **サマリーだけで判断可能に**: タイトルだけでなく2-3行の説明を含める
2. **Progressive Disclosure**: 4層で段階的に情報を開示
3. **技術詳細の分離**: ソースコードはAppendixへ
4. **視覚的な情報提示**: ダイアグラム・表を積極活用
5. **ナビゲーションの充実**: 概要と詳細の両方へリンク
