# IPA非機能要求グレード → 運用設計 マッピング表

IPA「非機能要求グレード2018」の6カテゴリと、運用設計書の各セクションとの対応関係を定義します。
全項目はIPA公式の4階層ID（カテゴリ記号.第1レベル.第2レベル.第3レベル、例: A.1.3.2）で参照します。

## 基本マッピング構造

```text
IPA非機能要求グレード（入力）
        ↓
[マッピング変換]
        ↓
運用設計書セクション（出力）
```

## 重複項目（○マーク）の処理規則

IPA活用シートには複数カテゴリに同一要件が記載される「重複項目（○マーク）」があります。

| 重複ペア | 処理方針 |
|---------|---------|
| A.1.1.1 ＝ C.1.1.1 | 値一致の場合はそのまま使用。不一致の場合はAカテゴリ（A.1.1.1）を優先し、ユーザーに確認 |
| A.1.1.3 ＝ C.2.1.1 | 同上（Aカテゴリ優先） |
| A.2.6.1 ＝ C.1.2.7 | 同上（Aカテゴリ優先） |
| A.2.6.2 ＝ C.1.2.1 | 同上（Aカテゴリ優先） |
| A.4.1.1 ＝ C.3.1.1 | 同上（Aカテゴリ優先） |

```text
C.6.x.x で「あり」が指定された場合:

```
